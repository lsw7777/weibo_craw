# 微博爬虫项目开发文档

## 1. 技术选型

### 后端

- `FastAPI`：提供 API 服务
- `requests`：请求微博接口
- `browser-cookie3`：读取本机浏览器中的微博登录 Cookie
- `Playwright`：执行微博账号搜索、关注、取关等网页交互
- `jieba`：中文分词，支持本地话题提取与观点归纳

### 前端

- `React`
- `Vite`
- 原生 `fetch`

## 2. 核心设计

### 2.1 登录态复用

项目不在后端再次实现微博登录，而是复用本机已经登录的浏览器状态：

1. `BrowserCookieProvider` 从 Edge 或 Chrome 中读取微博 Cookie
2. 内容抓取通过 `requests.Session` 携带 Cookie 访问微博接口
3. 关注 / 取关通过 `Playwright` 将 Cookie 注入浏览器上下文，再打开微博页面执行操作

兼容兜底：

- 如果浏览器 Cookie 数据库被 Windows 或浏览器自身锁定，可直接使用 `WEIBO_COOKIE_STRING`
- 这样不需要管理员权限，也不强依赖 `browser-cookie3` 成功访问原始 Cookie 文件
- 前端通过 `/api/auth/cookie/status` 检测登录态，通过 `/api/auth/cookie` 保存手工 Cookie

这套方式的优点：

- 不必额外维护登录流程
- 用户已经登录的微博账号可以直接使用
- 与“目前浏览器已经登录微博账号”的需求一致

### 2.2 数据抓取链路

抓取流程由 `WeiboCrawlerService` 负责：

1. 解析输入账号 URL，提取 UID
2. 请求 `profile/info` 获取账号基础信息
3. 请求 `statuses/mymblog` 分页获取微博列表
4. 按 `max_posts`、`start_time`、`end_time` 过滤微博
5. 对每条微博请求 `buildComments` 抓取评论
6. 解析微博与评论中的图片链接
7. 可选下载图片到 `backend/data/downloads/`
8. 调用 `ContentAnalyzer` 生成发博和评论分析结果
9. 将结构化结果导出到 `backend/data/exports/`

### 2.3 账号搜索与关注管理

该能力由 `WeiboFollowService` 负责：

- 搜索：打开 `https://s.weibo.com/user?q=关键词`，提取账号链接和名称
- 批量关注：依次打开用户主页，定位“关注”按钮并点击
- 批量取关：定位“已关注 / 互相关注”按钮，尝试执行取消关注并确认

说明：

- 微博页面结构存在变更风险，因此按钮定位采用了较宽松的文本匹配策略
- 如果后续页面结构变化，优先修改 `follow_service.py` 内的 `_find_follow_button` 和 `_find_clickable_by_text`
- 如果自动登录态注入失败，优先检查 `WEIBO_COOKIE_STRING` 是否配置

### 2.4 摘要分析

默认分析器是轻量本地实现：

- 使用 `jieba` 分词统计高频话题
- 使用正负面词表判断整体态度
- 从原始文本中抽取代表性句子作为观点样本

输出内容包括：

- `topics`
- `viewpoints`
- `sentiment`
- `summary`

如果要升级成 LLM 总结，建议新增一个 `LLMAnalyzer`，并在 `ContentAnalyzer` 前增加策略分发层。

## 4. API 设计

### `GET /api/health`

用途：健康检查

返回：

```json
{
  "status": "ok"
}
```

### `GET /api/auth/cookie/status`

用途：检测后端当前是否能读取微博登录态。

返回示例：

```json
{
  "configured": true,
  "readable": true,
  "source": "manual",
  "cookie_count": 12,
  "message": "已读取微博登录态。"
}
```

### `POST /api/auth/cookie`

用途：由前端保存手工 Cookie 到 `backend/.env`，并立即更新当前后端进程配置。

请求体示例：

```json
{
  "cookie_string": "SUB=...; XSRF-TOKEN=...; ..."
}
```

### `POST /api/scrape`

用途：批量抓取微博内容、评论、图片并生成分析

请求体示例：

```json
{
  "accounts": [
    "https://weibo.com/u/3074452897",
    "https://weibo.com/u/1917281600"
  ],
  "max_posts": 20,
  "start_time": "2026-04-01T00:00:00",
  "end_time": "2026-04-30T23:59:59",
  "fetch_comments": true,
  "max_comments_per_post": 20,
  "download_images": true,
  "save_json": true
}
```

### `GET /api/accounts/search?q=关键词&limit=10`

用途：搜索微博账号

### `POST /api/accounts/follow`

用途：批量关注或取关

请求体示例：

```json
{
  "action": "follow",
  "targets": [
    "https://weibo.com/u/3074452897",
    "https://weibo.com/u/1917281600"
  ]
}
```

## 5. 可扩展点

### 5.1 更强的账号搜索

目前搜索结果解析来自微博搜索页 DOM。可扩展方向：

- 增加用户主页信息回填
- 增加认证信息、粉丝数、简介提取
- 增加搜索结果缓存

### 5.2 更稳的抓取能力

当前微博接口依赖网页登录态。可扩展方向：

- 增加请求失败重试
- 增加限速与断点续抓
- 支持转发微博、视频、置顶微博的特殊处理
- 记录接口原始响应，便于排查微博字段变化

### 5.3 更强的分析能力

建议后续引入可配置的分析后端：

- 本地规则模式
- OpenAI / 兼容模型模式
- 私有部署模型模式

## 6. 已知限制

- 微博网页和接口结构可能变化，需定期维护
- 如果微博触发风控或验证码，自动抓取和关注操作会失败
- 评论分页字段和图片字段在不同类型微博上可能存在差异
- 当前输入账号主要支持 UID 或 `https://weibo.com/u/数字UID` 形式
- 在部分 Windows 环境中，浏览器原始 Cookie 文件会被锁定，因此自动读取可能失败，此时应改用 `WEIBO_COOKIE_STRING`

## 7. 调试建议

### 后端

- 先访问 `http://127.0.0.1:8000/api/health`
- 如果抓取失败，先看是否报 `403`
- 如果关注功能失败，优先检查 Playwright 浏览器安装情况

### 前端

- 开发环境默认请求 `http://127.0.0.1:8000`
- 如需改地址，可设置 `VITE_API_BASE_URL`
